### 总体空间：情绪洗衣房

整个项目只有一个主场景：

> 一个洗衣房背景，里面有洗衣机、脏衣篓、NPC、小票机/柜台、衣柜/收纳柜等元素。

用户不是“跳页面”，而是在这个空间里完成完整流程。

### 1. 洗衣房初始状态

用户进入后看到完整洗衣房。

画面中心是洗衣机，整个画面氛围轻松一点。

文案：

> 欢迎来到情绪洗衣房
>
> 今天有什么心事需要洗一洗？

按钮：

> 开始洗衣

点击后，不是跳到一个完全新的输入页，而是 **镜头移动/聚焦到脏衣篓区域** 。

### 2. 输入情绪动画：聚焦脏衣篓

镜头慢慢推近到左下角的脏衣篓。

脏衣篓旁边出现输入框。

文案可以是：

> 把今天的情绪脏衣服丢进来吧。

输入框 placeholder：

> 比如：我今天很烦，感觉什么都做不好……

如果用户输入太短，比如“烦”“累”“不开心”，可以先不立刻进入清洗，而是让 NPC 轻轻追问一句：

> 1、怎么了，发生什么事了？
>
> 或者
>
> 2、这种感觉真的很累，觉得哪里都卡住了。
>
> 是有什么具体的事让你觉得做不好，还是最近整体状态不太对劲？

但这里不要太强制。可以给两个按钮：

> 再说一点
>
> 直接开洗

这样用户不会觉得被审问

### 3. 投衣动画：把情绪词变成衣服

用户确认后，输入内容变成一件皱巴巴的小衣服/布团/纸团。

| 输入识别情绪                | 最终显示衣服 |
| --------------------------- | ------------ |
| 焦虑                        | 焦虑衣服     |
| 愤怒                        | 愤怒袜子     |
| 疲惫                        | 疲惫外套     |
| 委屈 / 难过 / 想哭          | 委屈毛衣     |
| 迷茫 / 无助 / 不知道怎么办  | 迷茫围巾     |
| 孤独 / 被忽略 / 没人懂      | 孤独手套     |
| 崩溃 / 发疯 / 破防          | 崩溃睡衣     |
| 内耗 / 纠结 / 自责 / 想太多 | 叠加线团效果 |

#### 判断优先级

| 优先级 | 情绪      | 显示规则                     |
| ------ | --------- | ---------------------------- |
| 1      | 崩溃/发疯 | 只要出现强崩溃词，就优先显示 |
| 2      | 愤怒      | 出现明显愤怒、攻击、不爽表达 |
| 3      | 委屈      | 出现想哭、被误会、没人理解   |
| 4      | 焦虑      | 出现担心、紧张、压力、害怕   |
| 5      | 疲惫      | 出现累、困、没力气、熬夜     |
| 6      | 迷茫      | 出现不知道怎么办、没方向     |
| 7      | 孤独      | 出现一个人、没人陪、被忽略   |
| 8      | 内耗      | 不作为主衣服，作为叠加效果   |

输入框里的文字 → 缩成一件“脏衣服” → 掉进脏衣娄

下一步转移视角自动移步到预洗台

### 4.预洗台：涂肥皂，识别情绪污渍

#### 场景聚焦

镜头从脏衣篓移动到旁边的 **预洗台** 。

脏衣服被摊开，上面有几块模糊污渍。

#### 用户看到什么

衣服上有污渍，但一开始看不清具体是什么。

旁边有肥皂 / 小刷子。

NPC 提示：

> 先看看这件衣服脏在哪里。

#### 用户做什么

用户拖动肥皂或刷子，在污渍上涂抹。

#### 发生什么

污渍逐渐显现为情绪标签，比如：

> 焦虑 / 委屈 / 疲惫 / 自责 / 烦躁

#### 这一步的意义

不是立刻解决情绪，而是先帮助用户看清楚：

> 我到底在难受什么？

#### 这一步的产出

> 情绪污渍标签。


#### 进度条设计

进度条放置在页面顶部，用来展示当前衣服的 **清洗进度** 。

进入清洗页后，系统会根据用户输入识别出主要情绪，并在进度条旁显示对应情绪，例如：

> 当前待清洗：焦虑
>
> 清洗进度：0%

用户通过涂抹肥皂、擦洗污渍等操作，让衣服上的情绪痕迹逐渐变淡。随着用户持续操作，进度条同步增长。

当进度条达到 100% 后，系统自动触发下一步：

> 衣服被放入洗衣机
>
> 页面视角从衣服区域平滑移动到洗衣机区域
>
> 进入滚筒清洗环节

这样用户能明确知道：

**我正在把这件情绪衣服洗干净，而且每次操作都有反馈。**

---

#### NPC 停滞提示

当用户一段时间没有操作，或者进度条没有明显变化时，NPC 可以给出轻提示，引导用户继续互动。

##### 可爱版

> 这里还有一点脏脏情绪，戳戳它！

> 诶？这件衣服好像还想再洗一会儿。

> 泡泡不够啦，再抹一点肥皂吧！

> 快好了快好了，就差最后一点点！

##### 温柔版

> 再轻轻搓一搓，它快变干净啦。

> 这块情绪污渍有点顽固，我们慢慢来。

> 不着急，先从最明显的地方洗起。

> 你已经洗掉一部分了，再坚持一下。

##### 鼓励版

> 你正在把它洗干净，已经有变化了。

> 这不是一下子就能洗掉的，但你已经开始处理它了。

> 别急，每搓掉一点，心事就轻一点。

> 这件衣服已经不像刚才那么沉了。
